# Delinea Secret Server Break-Glass Ansible Automation Walkthrough

Congratulations! The `delinea-breakglass-ansible` repository has been fully upgraded to natively accept decoupled JSON inputs. 

## JSON-Based Architecture Implementation

> [!NOTE]
> Based on your latest feedback, the system is now completely untethered from static internal variables mappings for `ad_groups` and `host_short_names`. 

### Fully Dynamic Target Handling 
The parameters map automatically inside the playbook:
- It processes external FQDN strings (e.g. `host_fqdn: "server01.corp.local"`) and logically slices the structural name mapping logic for internally storing inside Delinea (`server01`).
- It processes complex structures natively separating Comma-Space formats via explicit `split(',') | map('trim')`.

### Multi-Group Lookup Processing (`AD_Lookup` module)
The playbook no longer searches for a strict single group, but natively pipelines a list of names passed into `api/v1/groups` iteration.
- Dynamically compiles a resulting `assigned_group_ids` matrix.
- Stops and outputs error traces predictably if **any** requested group fails `domainId` validations.

### Parameterized Multi-Entity RBAC Bindings (`Permissions` module)
Both Secret Permissions and Folder Permissions successfully consume the compiled list array generated by the AD Lookup iteration, looping cleanly by executing sequential iterations against the Secret Server architecture.

## How to Trigger

You can now trigger the deployment seamlessly through either strategy.

### Method 1. Execute Utilizing External JSON 
Execute the target architecture strictly driving variables via JSON mapping to eliminate code coupling dependencies.

```bash
ansible-playbook \
   delinea-breakglass-ansible/playbooks/onboard_breakglass.yml \
   -i localhost, \
   -e "@delinea-breakglass-ansible/examples/input.json"
```

### Method 2. Integration via Core Playbooks (Wrapper Call)
Integrate this playbook into a larger onboarding strategy directly utilizing core system dictionary derivations:

```yaml
# caller_playbook.yml
---
- name: Wrapper Playbook to Call Delinea Onboarding
  ansible.builtin.import_playbook: ../playbooks/onboard_breakglass.yml
  vars:
    host_fqdn: "dbserver02.corp.local"
    template_id: 6001
    ad_groups: "BreakGlass_Admins, Database_Admins"
    domain: "corp.local"
    username: "breakglass_dbserver02"
    password: "AnotherSecurePassword123!"
```
